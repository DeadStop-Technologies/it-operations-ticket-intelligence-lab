# Interview Talk Track

## 30-second version

I built an IT Operations Ticket Intelligence Lab to show how I use data to improve support operations. It takes ticket data, loads it into CSV and SQLite, runs analysis with Python and SQL, and produces an executive-style summary. The goal is to identify repeat issues, reduce preventable tickets, improve SLA performance, and help leadership make better decisions.

## 60-second version

In my previous work, I learned that customers do not only need tickets closed. They need patterns found, repeat issues reduced, and clear communication from someone who can connect technical work to business impact. This project demonstrates that mindset. I created synthetic ticket data, analyzed SLA performance, escalations, root causes, and ticket categories, then turned the results into a report a manager or customer could understand. It shows Python, SQL, GitHub, documentation, and operational thinking in one project.

## Strong answer when asked why GitHub matters

GitHub gives me a place to prove what I can do instead of only saying it on a resume. A hiring manager can see my documentation, my scripts, my SQL, my project structure, and how I explain technical work in business terms.

## What to say if you are not a developer

I am not presenting this as a senior software engineering project. I built it as an IT operations and customer success project. It shows that I can use technical tools to solve business problems, automate manual work, and communicate findings clearly.
